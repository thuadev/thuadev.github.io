Life of a Request

原文：https://www.envoyproxy.io/docs/envoy/latest/intro/life_of_a_request

下面我们将描述通过特使代理的请求在生命中发生的事件。我们首先描述特使如何融入请求的请求路径，然后描述从下游到特使代理的请求到达后发生的内部事件。我们跟踪请求直到相应的调度上游和响应路径。

术语

Cluster：具有一组endpoints的逻辑服务，envoy将请求转发到这些endpoints上。

Downstream：连接到Envoy的实体。 这可能是本地应用程序（在Sidecar模型中）或网络节点。 在非Sidecar模型中，这是一个远程客户端。

Endpoints: 实现逻辑服务的网络节点。 它们被分组。 群集中的端点在Envoy代理的上游。

Filter: 连接或请求处理管道中的模块，用于提供请求处理的某些方面。 Unix的一个类比是小型实用程序（过滤器）与Unix管道（过滤器链）的组合。

Filter chain: 一系列的filter

Listeners： Envoy模块，负责绑定到IP /端口，接受新的TCP连接（或UDP数据报），并编排请求处理的下游方面。

Upstream： 转发服务请求时，Envoy连接到的端点（网络节点）。 这可能是本地应用程序（在Sidecar模型中）或网络节点。 在非Sidecar模型中，这对应于远程后端。

Network topology

systemd

发表于 2020-08-01

收藏：https://cloud.tencent.com/developer/article/1516125

Systemctl是一个systemd工具，主要负责控制systemd系统和服务管理器。

Systemd是一个系统管理守护进程、工具和库的集合，用于取代System V初始进程。Systemd的功能是用于集中管理和配置类UNIX系统。

查看systemd相关进程

pid 1的进程，是系统启动后的第一个进程，也就是init进程

systemd unit

Systemd 可以管理所有系统资源，不同的资源统称为 Unit（单位）。

Systemd 支持的 12 种 Unit 文件类型
  • .automount：用于控制自动挂载文件系统，相当于 SysV-init 的 autofs 服务
  • .device：对于 /dev 目录下的设备，主要用于定义设备之间的依赖关系
  • .mount：定义系统结构层次中的一个挂载点，可以替代过去的 /etc/fstab 配置文件
  • .path：用于监控指定目录或文件的变化，并触发其它 Unit 运行
  • .scope：这种 Unit 文件不是用户创建的，而是 Systemd 运行时产生的，描述一些系统服务的分组信息
  • .service：封装守护进程的启动、停止、重启和重载操作，是最常见的一种 Unit 文件
  • .slice：用于表示一个 CGroup 的树，通常用户不会自己创建这样的 Unit 文件
  • .snapshot：用于表示一个由 systemctl snapshot 命令创建的 Systemd Units 运行状态快照
  • .socket：监控来自于系统或网络的数据消息，用于实现基于数据自动触发服务启动
  • .swap：定义一个用户做虚拟内存的交换分区
  • .target：用于对 Unit 文件进行逻辑分组，引导其它 Unit 的执行。它替代了 SysV-init 运行级别的作用，并提供更灵活的基于特定设备事件的启动方式
  • .timer：用于配置在特定时间触发的任务，替代了 Crontab 的功能
Systemd 目录

Unit 文件按照 Systemd 约定，应该被放置指定的三个系统目录之一中。这三个目录是有优先级的，如下所示，越靠上的优先级越高。因此，在三个目录中有同名文件的时候，只有优先级最高的目录里的那个文件会被使用。

  • /etc/systemd/system：系统或用户自定义的配置文件
  • /run/systemd/system：软件运行时生成的配置文件
  • /usr/lib/systemd/system：系统或第三方软件安装时添加的配置文件。

使用fiddler抓包java内部请求

发表于 2020-07-28 更新于 2020-07-29

fiddler作为抓包利器，能够非常方便的对http请求数据包进行抓取，主要原理是为浏览器和本地http请求设置网络代理。

网络代理的地址是：127.0.0.1 8888

1）对于本地启动的localhost服务，在抓包的时候，发现请求在fiddler中没有拦截到

解决办法：

修改请求地址为：localhost.fiddler:8080

localhost加上.fiddler；请求被正常抓包

2）对于java程序内部发起的http请求，由于不会走http proxy，所以没有办法被fiddler抓包

比如feignclient或者httpclient发出的请求，调试的时候也是希望对这些请求的参数进行抓包测试的。

解决办法：通过 jvm 提供的参数，设置代理的ip和端口

设置方法：

1：java -DproxySet=true -DproxyHost=127.0.0.1 -DproxyPort=8888 MyApp

2：通过idea设置启动参数
